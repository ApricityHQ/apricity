\chapter{Testing}

\section{Testing Methodology}

Testing was carried out to verify that the system behaves correctly under different conditions. Two general approaches were used.

White-box testing was applied to individual components where the internal logic was known. This includes functions like text chunking, PDF extraction, and authentication checks, which were tested directly by calling them with controlled inputs.

Black-box testing was applied at the pipeline level, where the system was given a startup pitch and the outputs were checked without considering internal steps. This covers the multi-agent workflow and database operations.

The following testing types were applied:

\begin{enumerate}
    \item \textbf{Unit Testing:} Individual backend functions were tested in isolation with no external API calls or LLM usage.

    \item \textbf{Integration Testing:} Multiple components were tested together, including the full agent pipeline, RAG retrieval, and database operations.
\end{enumerate}
